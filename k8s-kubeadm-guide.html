SETUP PROGRESS 0 / 5 phases
1
Install required tools on your local machine
▾

You need Terraform, AWS CLI, and an SSH key pair before running anything.

terraform >= 1.5 installed
aws cli v2 installed & configured (aws configure)
SSH key exists at ~/.ssh/id_rsa.pub
AWS IAM user has EC2, VPC permissions
# Verify tools
terraform --version
aws --version
cat ~/.ssh/id_rsa.pub

# If no SSH key exists, generate one
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa
2
Configure terraform.tfvars
▾

Copy the example file and fill in your values. The most important thing is to restrict SSH to your IP.

cp terraform.tfvars.example terraform.tfvars

# Get your current public IP
curl https://checkip.amazonaws.com
# Use it as: allowed_ssh_cidr = "YOUR_IP/32"
⚠️ Always set allowed_ssh_cidr to your IP/32 — never leave it as 0.0.0.0/0 for real usage.
1
terraform init & plan
▾

Initialise Terraform and preview what it will create: 1 VPC, 1 subnet, 2 security groups, 2 EC2 instances, 1 key pair.

terraform init
terraform plan
💡 You should see 10 resources to add. Review them before applying.
2
terraform apply
▾

This provisions all AWS infrastructure. Both EC2 instances will automatically start running the bootstrap scripts via user_data.

terraform apply
# Type 'yes' to confirm

# After apply completes, note the outputs:
# master_public_ip = "13.x.x.x"
# worker_public_ip = "13.x.x.y"
✅ Terraform done! The cloud-init scripts are now running in the background on both nodes. Wait 3–5 minutes before SSHing in.
1
Wait for master bootstrap to complete
▾

SSH into the master and watch the init log. The script installs containerd, kubeadm, runs kubeadm init, and installs Flannel CNI.

ssh -i ~/.ssh/id_rsa ec2-user@MASTER_IP

# Watch the bootstrap log
sudo tail -f /var/log/k8s-master-init.log

# ✅ Look for this line:
# "Master node ready!"
2
Verify master is healthy
▾
# On master node:
kubectl get nodes
# Expected: master   Ready   control-plane

kubectl get pods -n kube-system
# All pods should be Running

kubectl get pods -n kube-flannel
# Flannel pod should be Running
3
Get the worker join command
▾

The bootstrap script automatically saved the join command for you.

# On master node:
cat ~/worker-join-command.sh

# Output will look like:
# kubeadm join 10.0.1.x:6443 --token abc123.xyz \
#   --discovery-token-ca-cert-hash sha256:...

# Copy this entire command!
⚠️ Tokens expire after 24 hours. If it's expired, run: kubeadm token create --print-join-command
1
Wait for worker bootstrap, then join
▾

SSH into the worker, wait for packages to install, then run the join command from the master.

# New terminal — SSH into worker
ssh -i ~/.ssh/id_rsa ec2-user@WORKER_IP

# Watch worker bootstrap
sudo tail -f /var/log/k8s-worker-init.log
# Wait for: "Worker node packages ready!"

# Run the join command (paste from master)
sudo kubeadm join 10.0.1.x:6443 \
  --token abc123.xyz \
  --discovery-token-ca-cert-hash sha256:...
✅ If successful, you'll see: This node has joined the cluster
1
Verify both nodes are Ready
▾
# On master node:
kubectl get nodes -o wide

# Expected output:
# NAME     STATUS   ROLES           VERSION
# master   Ready    control-plane   v1.29.x
# worker   Ready    <none>          v1.29.x

Deploy a test workload
▾
# Deploy nginx
kubectl run nginx --image=nginx --port=80
kubectl expose pod nginx --type=NodePort --port=80

# Get the NodePort
kubectl get svc nginx

# Access in browser:
# http://WORKER_IP:<NodePort>
3
Copy kubeconfig locally (optional)
▾

Control your cluster from your local machine instead of SSHing each time.

# On your LOCAL machine:
scp -i ~/.ssh/id_rsa \
  ec2-user@MASTER_IP:~/.kube/config \
  ./kubeconfig

export KUBECONFIG=./kubeconfig
kubectl get nodes   # works from local!
